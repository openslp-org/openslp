New in 0.7.8

  • slpd now does a graceful shutdown on SIGTERM to deregister serviced with DAs
  • Considerable time spend using commercial debugging tools to remove ALL memory leaks from slpd and libslp.  Code should very clean.
  • IPC (via loopback) is now used by libslp to obtain DA information from slpd before multicasting
  • slpd running as a DA now sends passive DAAdverts
  • slpd running as an SA now performs active DA discovery
  • slpd running as an SA now re-registers all services with new DAs
  • Significant fixes were made to make slpd work properly on multi-homed machines
